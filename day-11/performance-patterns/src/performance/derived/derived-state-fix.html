Improve a Derived State Anti-Pattern

Based on hajdun/15-days-of-react-design-patterns·▶ Watch on YouTube (28:07)

1What is wrong here

❌
The anti-pattern: filtered is fully computed from items, but it is still stored in state and synchronized with useEffect. That duplicates data and causes an extra render cycle.
  Broken pattern
const [filtered, setFiltered] = useState([]);

useEffect(() => {
  setFiltered(items.filter(i => i.active));
}, [items]);

2Correct derived-state pattern

✅
Fix: derive the value directly during render. If a value can be computed from props or existing state, do not store it in its own state.
  Correct pattern
const filtered = items.filter(i => i.active);

3Repo version: cart total

The repo's derived/ example uses a cart total instead of a filtered list, but it demonstrates the same rule: derive it, don't sync it.

❌ Cart.jsx
import { useEffect, useState } from "react";

const Cart = ({ items }) => {
  const [total, setTotal] = useState(0);

  useEffect(() => {
    const sum = items.reduce((acc, item) => acc + item.price, 0);
    setTotal(sum);
  }, [items]);

  return <h2>Total: {total}</h2>;
};
✅ BetterCart.jsx
function BetterCart({ items }) {
  const total = items.reduce((acc, item) => acc + item.price, 0);

  return <h2>Total: {total}</h2>;
}

4Your task solved

  FilteredItems.jsx — solved
function FilteredItems({ items }) {
  const filtered = items.filter((item) => item.active);

  return (
    <ul>
      {filtered.map((item) => (
        <li key={item.id}>{item.name}</li>
      ))}
    </ul>
  );
}

export default FilteredItems;
💡
If filtering is expensive and items is large, you can use useMemo for performance. But useMemo is optional here — the real fix is removing unnecessary state.

6Why derived state is better

ApproachWhat happensResult
useState + useEffectRender once, then effect runs, then setState triggers another renderMore code, duplicated state, extra render
const filtered = ...Value is computed directly from current props during renderSimpler, always in sync, no extra state
useMemo(() => ..., [items])Still derived, but cached between rendersUseful only if computation is expensive
