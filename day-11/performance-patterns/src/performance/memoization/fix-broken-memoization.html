Fix Broken Memoization

Based on hajdun/15-days-of-react-design-patterns · ▶ Watch on YouTube (12:08)

1Why Child was re-rendering

❌
The bug: Even though Child is wrapped in React.memo(), clicking Increment causes it to re-render on every click — because the onClick prop is a new function reference created fresh on each render.
1

User clicks Increment

setCount() is called → React schedules a re-render of App.

2

App re-renders — the arrow function is recreated

Every time App renders, () => console.log("Child clicked") is a brand-new object in memory — a different reference than the previous render.

3

memo() sees a changed prop

React.memo() does a shallow comparison: old reference ≠ new reference → props "changed" → Child re-renders. The memo wrapper is defeated.

2The broken code

  ChildDemo.jsx — ❌ broken
import { useState } from "react";
import Child from "./Child";

const ChildDemo = () => {
  const [count, setCount] = useState(0);

  // ❌ New function reference created on EVERY render
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      <Child onClick={() => console.log("Child clicked")} />
    </div>
  );
};
  Child.jsx
import { memo } from "react";

// memo() wraps the component — but it only helps if props are stable
const Child = memo(function Child({ onClick }) {
  console.log("Child Rendered");
  return <button onClick={onClick}>Click Me</button>;
});

export default Child;

3The fix — useCallback()

✅
Solution: Wrap the handler with useCallback() so React returns the same function reference between renders. Now memo()'s shallow comparison sees identical props → Child skips re-rendering.
  ChildDemo.jsx — ✅ fixed (from repo)
import { useCallback, useState } from "react";
import Child from "./Child";

const ChildDemo = () => {
  const [count, setCount] = useState(0);

  // ✅ Stable reference — only recreated if deps change
  const handleClick = useCallback(() => {
    console.log("Child Clicked!");
  }, []); // empty deps — function never needs to change

  console.log("App Rendered");

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
      </* onClick is now the same reference every render */>
      <Child onClick={handleClick} />
    </div>
  );
};
💡
The dependency array [] means "this function doesn't need any values from the component scope — never recreate it." If your handler uses state or props, list them: useCallback(() => doThing(id), [id]).

4Before vs. after — side by side

❌ Before (broken)
<Child
  onClick={
    () => console.log("clicked")
    // new fn every render!
  }
/>
✅ After (fixed)
const handleClick = useCallback(
  () => console.log("clicked"),
  [] // stable reference
);
<Child onClick={handleClick} />

6Why each piece matters

Tool What it does Without it
React.memo() Skips re-rendering Child if props are shallowly equal Child always re-renders with parent — memo is useless alone
useCallback() Caches the function reference across renders New reference every render → memo() sees "changed" prop → re-render
Dependency array [] Controls when the callback is recreated Wrong deps → stale closures or unnecessary recreations
